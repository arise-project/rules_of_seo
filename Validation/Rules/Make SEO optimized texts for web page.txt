Рекомендации по написанию SEO оптимизированных текстов


1. Прежде чем начинать писать текст для какой либо страницы нужно составить семантическое ядро, т.е. Подобрать ключевые слова и фразы актуальные именно для этой страницы, для этого Вам потребуется такой инструмент 
https://ads.google.com/aw/keywordplanner/home 


PS: для доступа к этому ресурсу Вам необходим аккаунт google и выполнить инструкции из этой статьи https://www.monsterinsights.com/how-to-use-google-keyword-planner/ 


   1. Для примера возьмем приложение /pub/conversion/pub-to-pdf , вводим в keywordplaner   предположительный топовый запрос (для этого приложения это будет “PUB to PDF”) и получаем следующую картину
  

В данном приложении к семантическому ядру можно отнести фразы с наибольшим числом запросов в месяц. ВАЖНО! Анализ нужно проводить по локали English и без учета региона (справа вверху на картинке).


   2. URL для приложения должен так или иначе содержать топовый запрос из Вашего семантического ядра. Для данного примера приложения Я рекомендую использовать один из следующих URLs
/pub/convert/pub-to-pdf
/pub/convert-pub-to-pdf
/pub/convert/publisher-to-pdf
/pub/convert-publisher-to-pdf


   3. Определите основных конкурентов страницы для которой вы пишите текст, это в дальнейшем пригодится для понимания типа контента страницы


2. Заголовок Н1 - должен в точности соответствовать топовому запросу из семантического ядра без каких либо дополнений типа “online”, “free” и т.д.
3. Заголовок Н2 (текст под заголовком Н1)
   1. Должен быть оформлен тегом Н2
   2. Должен содержать одну или несколько фраз из семантического ядра, в идеале Н2 должен начинаться с фразы из семантического ядра.
   3. Длина заголовка Н2 должна быть в диапазоне 50 - 90 символов
   4. Если по какой то причине нужно расширить информацию в заголовке Н2, лучше разбить этот текст на два блока отдельный небольшой блок поместить в Н2 и отдельный дополнительный блок <p></p> сразу под Н2.
4. Page Title
   1. Должен содержать одну из топовых фраз из семантического ядра, но не самую короткую, для нашего примера можно было бы написать “Convert PUB to PDF online”
   2. Рекомендуемая длина 40-60 символов
   3. Длина может быть и 20-30 символов, но тогда учитывайте что Гугл будет автоматически туда добавлять название домена или бренда
5. Page Description
   1. Должен содержать одну или несколько топовых фраз из семантического ядра.
   2. Длина не должна превышать 150 символов.
   3. Не стоит упоминать в этом поле бренд, например “Aspose.PUB”
   4. Для упрощения написания Description, можно ввести в поисковую строку гугла ваш основной ключевой запрос (например “PUB to PDF”), выделить несколько лидеров выдачи гугла, взять их Description и сформировать свой, компонуя несколько разных.
6. Раздел HowTo
   1. Title блока HowTo должен быть оформлен тегом Н2 или Н3, и содержать очевидный текст с использованием основного ключевого слова (например “How to convert PUB to PDF”)
   2. В строках должны быть описаны шаги что нужно сделать чтобы выполнить конвертацию в этом приложении Step-By-Step по возможности применяя фразы из семантического ядра
7. Раздел FAQ
   1. Title блока FAQ должен быть оформлен тегом Н2 или Н3, аналогично как и блок HowTo
   2. Для того чтобы понять какие вопросы стоит поместить в этот блок, необходимо в поисковой строке гугл ввести запрос похожий на “PUB to PDF FAQ”, вы увидите что то похожее на это 
  

Выберите несколько интересных вариантов (4-6) и разместите их на своей странице
8. Раздел Overview, один из наиболее важных разделов
   1. Этот размер должен содержать мини статью по тематике странице, вы можете написать тут мини инструкцию как пользоваться приложением, какие то особенности приложения, описание возможностей приложения и общие фразы. ВАЖНО! Этот блок должен быть достаточно информативным и полезным для потенциального пользователя, тут может быть информация из Wikipedia (перефразированная) или какого то стандарта типа ISO. Так же этот блок должен содержать фразы из семантического ядра.
   2. Количество ключевых слов в этом блоке не должно быть более 4-5 (для количества символов в этом блоке 1500 - 2000).
   3. Размер этого блока в идеале должен быть 2500 - 3000 символов. Но размер может быть и 1000-1500 символов, основное, что нужно помнить, что общий размер текста на вашей странице должен быть на 20-30% больше чем у конкурентов (см. п. 1.3.).
   4. ВАЖНО! Текст в этом блоке должен быть максимально уникальным (на 90-100%), уникальность текста вы можете проверить с помощью программы “ADVEGO PLAGIATUS” или аналогичных. Если после проверки Ваш текст оказался не достаточно уникальным, внесите правки по не уникальным фрагментам и повторно выполните проверку. При проверке на уникальность, в настройках программы исключите такие домены как aspose.app, aspose.com, groupdocs.com, groupdocs.app, aspose.cloud
   5. Иногда после валидации текста на уникальность вы можете обнаружить что текст уникален например на 10-20%, при этом вы уверены, что писали его самостоятельно, если внимательно посмотреть в отчете программы вы можете найти домены (страницы) с точной копией вашего текста, такое бывает, некоторые сайты клонируют наши страницы, поэтому обнаружив такой домен просто добавьте его в исключения и снова выполните проверку.
9. Микроразметка
   1. Убедитесь что микроразметка заполняется в полном объеме, на всех локалях и на всех sub-formats (Т.е. /pub/conversion, /pub/conversion/pub, /pub/conversion/pub-to-pdf), для этого используйте вот этот ресурс https://search.google.com/test/rich-results 
10. Локализация
   1. Для переводов на разные локали можно использовать сервис https://weblate.svc.conholdate.cloud 
   2. Перечень локалей рекомендуемых для имплементации можно позаимствовать у продуктов https://products.aspose.app/pdf/conversion или https://products.aspose.app/words/merger , внедрять еще большее количество локалей нужно с осторожностью,  т.к. большого эффекта это не даст, а работы нужно будет проделать много. Опытным путем были выявлены наиболее востребованные локали.
11. Ссылка на документацию docs.aspose.com или на Landing Pages products.aspose.com (сейчас более актуально)
   1. На каждой странице приложения Вы можете разместить ссылку на страницу документации соответствующего продукта и приложения, таким образом Вы улучшите рейтинг страницы документации и дадите им больше посетителей. 
Например на странице 
/pub/conversion/pub-to-pdf 
можно разместить ссылку 
docs.aspose.com/pub/net/pub-to-pdf/
на странице 
/pub/conversion/pub  ->
docs.aspose.com/pub/net/convert-pub-files/
На странице
/pub/conversion ->
docs.aspose.com/pub/net/product-overview/
Т.к. это общие страницы
/pub/conversion/pub-to-jpeg  ->
docs.aspose.com/pub/net/product-overview/
Т.к. В доке нет аналогичной страницы
12. Плотность ключевых слов
   1. Количество упоминаний бренда Aspose в тексте должно быть как можно меньше, в том числе в заголовках и кнопках, т.к. Таким образом слово Aspose становится ключевым и преобладающим, соответственно для гугла тематика страницы приложения становится не Конвертация, а Aspose + Конвертация что негативно влияет на рейтинг страницы
   2. Количество основных ключевых слов на странице (согласно семантического ядра) должно быть в диапазоне 5-10% с учетом всех заголовков, списков и метатегов, проверить можно http://simple-seo-tools.com/ru/densityanalyzer или https://eptimize.app/keywords-extractor или подобным инструментом.
